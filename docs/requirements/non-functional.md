## 4. セキュリティ

### 4.1 認証・認可

| 項目 | 要件 |
|-----|-----|
| パスワードポリシー | 最低8文字 |
| 多要素認証 | 【将来対応】 |
| セッション有効期限 | アクセストークン1時間、リフレッシュ7日 |
| ログイン試行制限 | 5回失敗で15分ロック（Supabase標準） |

### 4.2 データ保護

| データ種別 | 暗号化 | 保持期間 |
|-----------|-------|---------|
| パスワード | bcrypt（Supabase Auth） | - |
| 個人情報 | RLS + TLS | 退会後30日 |
| 学習データ | RLS + TLS | 退会後30日 |

### 4.3 通信

| 項目 | 要件 |
|-----|-----|
| プロトコル | HTTPS（TLS 1.3） |
| 証明書 | Vercel / Supabase 自動管理 |

### 4.4 脆弱性対策

| 脅威 | 対策 |
|-----|-----|
| XSS | CSP, React自動エスケープ |
| CSRF | SameSite Cookie |
| SQLインジェクション | Supabaseパラメータ化クエリ |
