## 概要

ユーザーがタグを一覧・作成・編集・削除できるようにする。タグはカードの分類に使用され、フィルタリングの基準となる。

## ユーザーフロー

### タグ一覧表示
1. **[トリガー]** ユーザーが「タグ管理」画面を開く
2. **[システム]** ユーザーが作成したタグ一覧を表示
3. **[完了]** タグ名、紐付けカード数が表示される

### タグ作成
1. **[トリガー]** ユーザーが「タグ追加」ボタンをタップ
2. **[システム]** タグ名入力フォームを表示
3. **[ユーザー]** タグ名を入力
4. **[システム]** タグを作成
5. **[完了]** 一覧に新しいタグが追加される

### タグ編集
1. **[トリガー]** ユーザーがタグをタップ
2. **[システム]** タグ編集フォームを表示
3. **[ユーザー]** タグ名を変更
4. **[システム]** タグを更新
5. **[完了]** 変更が反映される（紐付けカードにも反映）

### タグ削除
1. **[トリガー]** ユーザーがタグの削除アイコンをタップ
2. **[システム]** 削除確認ダイアログを表示
3. **[ユーザー]** 「削除」をタップ
4. **[システム]** タグと紐付けを削除（カード自体は残る）
5. **[完了]** 一覧からタグが消える

## 入力/出力

| 種別 | 項目名 | 型 | 必須 | 説明 | 制約・バリデーション |
|-----|-------|---|-----|-----|-------------------|
| 入力 | name | string | Yes | タグ名 | 1-30文字 |
| 入力 | color | string | No | タグの色 | Hex形式 (#RRGGBB) |
| 出力 | tag | Tag | - | タグ情報 | id, name, color, card_count |

## ビジネスルール

| ID | ルール | 条件 | 結果 |
|----|-------|-----|-----|
| BR-F017-01 | 名前の一意性 | 同一ユーザー内 | 同名タグは作成不可 |
| BR-F017-02 | タグ削除時 | カードとの紐付け | 紐付けのみ削除（カードは残る） |
| BR-F017-03 | タグ名変更時 | 既存の紐付け | 紐付けは維持される |
| BR-F017-04 | 未使用タグ | 紐付けカード0件 | 削除可能、警告なし |
| BR-F017-05 | 使用中タグ | 紐付けカード1件以上 | 削除時に確認ダイアログ |

## データモデル

### Tag
```typescript
type Tag = {
  id: string        // UUID
  user_id: string   // 所有者
  name: string      // タグ名
  color: string     // 表示色 (Hex)
  created_at: string
  updated_at: string
}
```

## エラーケース

| ID | 条件 | エラーメッセージ | 対処 |
|----|-----|----------------|-----|
| E-F017-01 | タグ名が空 | タグ名を入力してください | フォーム再入力 |
| E-F017-02 | タグ名が長すぎる | タグ名は30文字以内にしてください | フォーム再入力 |
| E-F017-03 | タグ名重複 | このタグ名は既に使用されています | 別の名前を入力 |
| E-F017-04 | 保存失敗 | タグの保存に失敗しました | リトライ |

## 受け入れ条件（AC）

- [ ] AC-01: タグ一覧を表示できること
- [ ] AC-02: 各タグに紐付けられているカード数が表示されること
- [ ] AC-03: 新しいタグを作成できること
- [ ] AC-04: タグ名を編集できること
- [ ] AC-05: タグを削除できること
- [ ] AC-06: タグを削除しても紐付いていたカードは残ること
- [ ] AC-07: 同名のタグは作成できないこと

## 技術仕様

### Supabase連携
```typescript
// タグ一覧取得（カード数付き）
const { data: tags } = await supabase
  .from('tags')
  .select(`
    *,
    card_count:card_tags(count)
  `)
  .eq('user_id', userId)
  .order('name')

// タグ作成
const { data, error } = await supabase
  .from('tags')
  .insert({ user_id: userId, name, color })
  .select()
  .single()

// タグ更新
const { error } = await supabase
  .from('tags')
  .update({ name, color, updated_at: new Date().toISOString() })
  .eq('id', tagId)
  .eq('user_id', userId)

// タグ削除（card_tagsはカスケード削除）
const { error } = await supabase
  .from('tags')
  .delete()
  .eq('id', tagId)
  .eq('user_id', userId)
```

### RLS
```sql
CREATE POLICY "Users can manage their own tags"
ON tags FOR ALL
USING (auth.uid() = user_id);
```
