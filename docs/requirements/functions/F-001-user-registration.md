# F-001: ユーザー登録

> 関連: [ビジネス要件](../business-requirements.md) | [機能一覧](./_index.md)

## 基本情報

| 項目 | 内容 |
|-----|-----|
| 機能ID | F-001 |
| 優先度 | P0 |
| 関連US | F-001 |
| ステータス | Draft |

## 概要

ユーザーがメールアドレスとパスワードで新規アカウントを作成し、学習データを保存・同期できるようにする。Supabase Authを使用した認証基盤。

## ユーザーフロー

1. **[トリガー]** ユーザーが「新規登録」ボタンをタップ
2. **[システム]** 登録フォームを表示（メールアドレス、パスワード、パスワード確認）
3. **[ユーザー]** 必要情報を入力し「登録」ボタンをタップ
4. **[システム]** 入力値をバリデーション
5. **[システム]** Supabase Authでアカウント作成
6. **[システム]** 確認メールを送信
7. **[ユーザー]** メール内のリンクをクリック
8. **[システム]** メールアドレスを確認済みに更新
9. **[完了]** ログイン状態でホーム画面へ遷移

## 入力/出力

| 種別 | 項目名 | 型 | 必須 | 説明 | 制約・バリデーション |
|-----|-------|---|-----|-----|-------------------|
| 入力 | email | string | Yes | メールアドレス | RFC 5322準拠、最大254文字 |
| 入力 | password | string | Yes | パスワード | 8文字以上、英数字混合必須 |
| 入力 | passwordConfirm | string | Yes | パスワード確認 | passwordと一致 |
| 出力 | user | User | - | 作成されたユーザー情報 | id, email, created_at |
| 出力 | session | Session | - | 認証セッション | access_token, refresh_token |

## ビジネスルール

| ID | ルール | 条件 | 結果 |
|----|-------|-----|-----|
| BR-F001-01 | メールアドレス重複禁止 | 既に登録済みのメールアドレスの場合 | エラーを返す（セキュリティ上「既に登録済み」とは明示しない） |
| BR-F001-02 | パスワード強度要件 | 8文字未満または英数字混合でない場合 | バリデーションエラー |
| BR-F001-03 | メール確認必須 | メール未確認状態 | ログイン不可（確認メール再送可能） |

## 状態遷移

| 現在の状態 | イベント | 次の状態 | 備考 |
|-----------|---------|---------|-----|
| 未登録 | 登録フォーム送信 | 確認待ち | 確認メール送信済み |
| 確認待ち | メールリンククリック | 登録完了 | ログイン可能 |
| 確認待ち | 24時間経過 | リンク期限切れ | 再送可能 |

## エラーケース

| ID | 条件 | エラーメッセージ | 対処 |
|----|-----|----------------|-----|
| E-F001-01 | メールアドレス形式不正 | 有効なメールアドレスを入力してください | フォーム再入力 |
| E-F001-02 | パスワードが要件を満たさない | パスワードは8文字以上、英数字を含める必要があります | フォーム再入力 |
| E-F001-03 | パスワード確認が一致しない | パスワードが一致しません | フォーム再入力 |
| E-F001-04 | メールアドレス重複 | アカウントを作成できませんでした。再度お試しください | フォーム再入力（既存ユーザー情報は漏らさない） |
| E-F001-05 | 確認メール送信失敗 | メール送信に失敗しました。再度お試しください | リトライ |

## 受け入れ条件（AC）

- [ ] AC-01: 有効なメールアドレスとパスワードで登録すると、確認メールが送信されること
- [ ] AC-02: 確認メールのリンクをクリックすると、アカウントが有効化されること
- [ ] AC-03: メール未確認状態ではログインできないこと
- [ ] AC-04: 重複メールアドレスで登録しようとした場合、汎用エラーメッセージが表示されること
- [ ] AC-05: パスワードが要件を満たさない場合、具体的なエラーメッセージが表示されること
- [ ] AC-06: 登録完了後、自動的にログイン状態になりホーム画面に遷移すること

## 画面要件

### 登録フォーム
- メールアドレス入力フィールド
- パスワード入力フィールド（表示/非表示トグル付き）
- パスワード確認入力フィールド
- パスワード要件の表示（8文字以上、英数字混合）
- 「登録」ボタン
- 「ログインはこちら」リンク
- Googleログインボタン（v1.1で追加予定）

### 確認メール送信完了画面
- 送信先メールアドレスの表示
- 「確認メールを再送」ボタン
- 「メールアドレスを変更」リンク

## 技術仕様

### Supabase Auth連携
```typescript
// 登録処理
const { data, error } = await supabase.auth.signUp({
  email,
  password,
  options: {
    emailRedirectTo: `${origin}/auth/callback`
  }
})
```

### メール確認コールバック
- `/auth/callback` でトークンを検証
- 検証成功後、セッションを確立してホームへリダイレクト

## 未確定事項

- 【要確認】パスワード要件を「大文字・小文字・数字・記号」まで厳しくするか
- 【仮定】メール確認リンクの有効期限は24時間と仮定
