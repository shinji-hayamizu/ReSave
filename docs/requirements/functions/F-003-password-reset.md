# F-003: パスワードリセット

> 関連: [ビジネス要件](../business-requirements.md) | [機能一覧](./_index.md)

## 基本情報

| 項目 | 内容 |
|-----|-----|
| 機能ID | F-003 |
| 優先度 | P0 |
| 関連US | F-003 |
| ステータス | Draft |

## 概要

パスワードを忘れたユーザーが、メールアドレス宛のリセットリンクを通じて新しいパスワードを設定できるようにする。

## ユーザーフロー

### リセットリンク送信
1. **[トリガー]** ユーザーが「パスワードを忘れた方」リンクをタップ
2. **[システム]** メールアドレス入力フォームを表示
3. **[ユーザー]** 登録済みメールアドレスを入力
4. **[システム]** リセットリンク付きメールを送信
5. **[完了]** 「メールを送信しました」画面を表示

### パスワード再設定
1. **[トリガー]** ユーザーがメール内のリセットリンクをクリック
2. **[システム]** トークンを検証し、パスワード再設定フォームを表示
3. **[ユーザー]** 新しいパスワードを入力（確認含む）
4. **[システム]** パスワードを更新
5. **[完了]** ログイン画面へ遷移（成功メッセージ付き）

## 入力/出力

| 種別 | 項目名 | 型 | 必須 | 説明 | 制約・バリデーション |
|-----|-------|---|-----|-----|-------------------|
| 入力 | email | string | Yes | 登録済みメールアドレス | RFC 5322準拠 |
| 入力 | newPassword | string | Yes | 新しいパスワード | 8文字以上、英数字混合必須 |
| 入力 | newPasswordConfirm | string | Yes | パスワード確認 | newPasswordと一致 |
| 出力 | success | boolean | - | 処理結果 | - |

## ビジネスルール

| ID | ルール | 条件 | 結果 |
|----|-------|-----|-----|
| BR-F003-01 | 未登録メールアドレス | 存在しないメールアドレスの場合 | 同じ成功メッセージを表示（情報漏洩防止） |
| BR-F003-02 | リセットリンク有効期限 | リンク発行から1時間経過 | リンク無効、再発行を促す |
| BR-F003-03 | リンク使い捨て | リセット完了後 | 同じリンクは使用不可 |
| BR-F003-04 | パスワード履歴 | 直近3回と同じパスワード | 設定不可（v1.1以降で検討） |

## 状態遷移

| 現在の状態 | イベント | 次の状態 | 備考 |
|-----------|---------|---------|-----|
| ログイン可能 | リセットリンク送信 | リセット待ち | メール送信済み |
| リセット待ち | リンククリック | パスワード入力待ち | トークン検証済み |
| パスワード入力待ち | 新パスワード設定 | ログイン可能 | 旧パスワード無効化 |
| リセット待ち | 1時間経過 | リンク期限切れ | 再発行可能 |

## エラーケース

| ID | 条件 | エラーメッセージ | 対処 |
|----|-----|----------------|-----|
| E-F003-01 | メールアドレス形式不正 | 有効なメールアドレスを入力してください | フォーム再入力 |
| E-F003-02 | リンク期限切れ | リンクの有効期限が切れています。再度リセットをお試しください | リセットリンク再送 |
| E-F003-03 | リンク使用済み | このリンクは既に使用されています | リセットリンク再送 |
| E-F003-04 | パスワード要件不足 | パスワードは8文字以上、英数字を含める必要があります | フォーム再入力 |
| E-F003-05 | パスワード確認不一致 | パスワードが一致しません | フォーム再入力 |

## 受け入れ条件（AC）

- [ ] AC-01: 登録済みメールアドレスを入力すると、リセットメールが送信されること
- [ ] AC-02: 未登録メールアドレスでも同じ成功メッセージが表示されること（情報漏洩防止）
- [ ] AC-03: リセットリンクから新しいパスワードを設定できること
- [ ] AC-04: パスワード設定後、旧パスワードでログインできなくなること
- [ ] AC-05: 1時間経過したリンクは無効となること
- [ ] AC-06: 一度使用したリンクは再使用できないこと

## 画面要件

### リセットリンク送信フォーム
- メールアドレス入力フィールド
- 「リセットリンクを送信」ボタン
- 「ログインに戻る」リンク

### 送信完了画面
- 「メールを送信しました」メッセージ
- メールが届かない場合の案内
- 「再送信」ボタン

### パスワード再設定フォーム
- 新しいパスワード入力フィールド
- パスワード確認入力フィールド
- パスワード要件の表示
- 「パスワードを変更」ボタン

## 技術仕様

### Supabase Auth連携
```typescript
// リセットメール送信
const { error } = await supabase.auth.resetPasswordForEmail(email, {
  redirectTo: `${origin}/auth/reset-password`
})

// パスワード更新
const { error } = await supabase.auth.updateUser({
  password: newPassword
})
```
