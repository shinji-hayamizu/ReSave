## 概要

ユーザーが作成済みのカードの内容（表面・裏面・タグ）を編集できるようにする。学習履歴（FSRSパラメータ）は保持される。

## ユーザーフロー

1. **[トリガー]** ユーザーがカード一覧または学習画面で「編集」をタップ
2. **[システム]** カード編集フォームを表示（現在の内容をプリセット）
3. **[ユーザー]** 表面・裏面・タグを編集
4. **[ユーザー]** 「保存」ボタンをタップ
5. **[システム]** カードを更新
6. **[完了]** 編集完了メッセージを表示、元の画面に戻る

## 入力/出力

| 種別 | 項目名 | 型 | 必須 | 説明 | 制約・バリデーション |
|-----|-------|---|-----|-----|-------------------|
| 入力 | cardId | string | Yes | 編集対象カードID | UUID形式 |
| 入力 | front | string | Yes | 表面（質問） | 1文字以上、最大2000文字 |
| 入力 | back | string | Yes | 裏面（答え） | 1文字以上、最大5000文字 |
| 入力 | tagIds | string[] | No | 付与するタグのID配列 | 既存タグのみ |
| 出力 | card | Card | - | 更新されたカード | 全フィールド |

## ビジネスルール

| ID | ルール | 条件 | 結果 |
|----|-------|-----|-----|
| BR-F014-01 | FSRS維持 | カード編集時 | FSRSパラメータは変更しない |
| BR-F014-02 | 所有者チェック | 他ユーザーのカード | 編集不可（404エラー） |
| BR-F014-03 | タグ更新 | tagIds変更時 | 既存の紐付けを削除し、新しい紐付けを作成 |
| BR-F014-04 | 更新日時 | 編集保存時 | updated_atを現在時刻に更新 |

## エラーケース

| ID | 条件 | エラーメッセージ | 対処 |
|----|-----|----------------|-----|
| E-F014-01 | カード不在 | カードが見つかりません | カード一覧に戻る |
| E-F014-02 | 権限なし | このカードを編集する権限がありません | カード一覧に戻る |
| E-F014-03 | 表面が空 | 表面（質問）を入力してください | フォーム再入力 |
| E-F014-04 | 裏面が空 | 裏面（答え）を入力してください | フォーム再入力 |
| E-F014-05 | 更新失敗 | カードの更新に失敗しました。再度お試しください | リトライ |

## 受け入れ条件（AC）

- [ ] AC-01: カードの表面・裏面を編集して保存できること
- [ ] AC-02: タグの追加・削除ができること
- [ ] AC-03: 編集後もFSRSパラメータ（学習履歴）が維持されること
- [ ] AC-04: 他ユーザーのカードは編集できないこと
- [ ] AC-05: 存在しないカードIDの場合、エラーが表示されること

## 画面要件

### カード編集フォーム
- 表面（質問）入力エリア（現在値をプリセット）
- 裏面（答え）入力エリア（現在値をプリセット）
- タグ選択UI（現在のタグをプリセット）
- 文字数カウンター
- 「キャンセル」ボタン（変更を破棄）
- 「保存」ボタン
- 「削除」ボタン（F-015へ）

## 技術仕様

### Supabase連携
```typescript
// カード更新
const { data, error } = await supabase
  .from('cards')
  .update({
    front,
    back,
    updated_at: new Date().toISOString()
  })
  .eq('id', cardId)
  .eq('user_id', userId)  // RLSに加えてアプリレベルでもチェック
  .select()
  .single()

// タグ更新（全削除→再作成）
await supabase
  .from('card_tags')
  .delete()
  .eq('card_id', cardId)

if (tagIds.length > 0) {
  await supabase
    .from('card_tags')
    .insert(tagIds.map(tagId => ({ card_id: cardId, tag_id: tagId })))
}
```

### RLS
```sql
CREATE POLICY "Users can update their own cards"
ON cards FOR UPDATE
USING (auth.uid() = user_id);
```
