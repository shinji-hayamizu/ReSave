# F-014: カード編集

> 関連: [ビジネス要件](../../business-requirements.md) | [機能一覧](../_index.md)

## 基本情報

| 項目 | 内容 |
|-----|-----|
| 機能ID | F-014 |
| 優先度 | P0 |
| 関連US | F-014, F-016 |
| ステータス | Draft |

## 概要

ユーザーが作成済みのカードの内容（テキスト・隠しテキスト・タグ・ソース・リピート設定）を編集できるようにする。学習履歴（復習パラメータ）は保持される。

## ユーザーフロー

1. **[トリガー]** ユーザーがホーム画面でカードの「編集」をタップ
2. **[システム]** カード入力画面を表示（現在の内容をプリセット）
3. **[ユーザー]** 以下の項目を編集:
   - テキスト（表面）
   - 隠しテキスト（裏面）
   - タグ
   - ソースURL
   - リピート設定
4. **[ユーザー]** 「保存」ボタンをタップ
5. **[システム]** カードを更新
6. **[完了]** ホーム画面に戻る

## 入力/出力

### 入力項目

| 種別 | 項目名 | 型 | 必須 | 説明 | 制約・バリデーション |
|-----|-------|---|-----|-----|-------------------|
| 入力 | cardId | string | Yes | 編集対象カードID | UUID形式 |
| 入力 | text | string | Yes | テキスト（カード表面） | 1文字以上、最大500文字 |
| 入力 | hiddenText | string | No | 隠しテキスト（カード裏面） | 最大2000文字 |
| 入力 | tagIds | string[] | No | 付与するタグのID配列 | 既存タグのみ |
| 入力 | sourceUrl | string | No | ソースURL | 有効なURL形式 |
| 入力 | repeatInterval | string | No | リピート設定 | 'default' or カスタム間隔 |

### 出力

| 種別 | 項目名 | 型 | 説明 |
|-----|-------|---|-----|
| 出力 | card | Card | 更新されたカード |

## ビジネスルール

| ID | ルール | 条件 | 結果 |
|----|-------|-----|-----|
| BR-F014-01 | 復習パラメータ維持 | カード編集時 | state, interval_index, next_review_date等は変更しない |
| BR-F014-02 | 所有者チェック | 他ユーザーのカード | 編集不可（404エラー） |
| BR-F014-03 | タグ更新 | tagIds変更時 | 既存の紐付けを削除し、新しい紐付けを作成 |
| BR-F014-04 | 更新日時 | 編集保存時 | updated_atを現在時刻に更新 |

## エラーケース

| ID | 条件 | エラーメッセージ | 対処 |
|----|-----|----------------|-----|
| E-F014-01 | カード不在 | カードが見つかりません | ホーム画面に戻る |
| E-F014-02 | 権限なし | このカードを編集する権限がありません | ホーム画面に戻る |
| E-F014-03 | テキストが空 | テキストを入力してください | フォーム再入力 |
| E-F014-04 | テキストが長すぎる | テキストは500文字以内で入力してください | フォーム再入力 |
| E-F014-05 | 無効なURL | 有効なURLを入力してください | URL修正 |
| E-F014-06 | 更新失敗 | カードの更新に失敗しました。再度お試しください | リトライ |

## 受け入れ条件（AC）

- [ ] AC-01: カードのテキスト・隠しテキストを編集して保存できること
- [ ] AC-02: タグの追加・削除ができること
- [ ] AC-03: ソースURLを編集できること
- [ ] AC-04: リピート設定を変更できること
- [ ] AC-05: 編集後も復習パラメータ（学習履歴）が維持されること
- [ ] AC-06: 他ユーザーのカードは編集できないこと
- [ ] AC-07: 文字数カウンター（0/500）が表示されること

## 技術仕様

### Supabase連携

```typescript
// カード更新
const { data, error } = await supabase
  .from('cards')
  .update({
    text,
    hidden_text: hiddenText || null,
    source_url: sourceUrl || null,
    repeat_intervals: repeatIntervals,
    updated_at: new Date().toISOString()
  })
  .eq('id', cardId)
  .eq('user_id', userId)
  .select()
  .single()

// タグ更新（全削除→再作成）
await supabase
  .from('card_tags')
  .delete()
  .eq('card_id', cardId)

if (tagIds.length > 0) {
  await supabase
    .from('card_tags')
    .insert(tagIds.map(tagId => ({ card_id: cardId, tag_id: tagId })))
}
```

### RLS

```sql
CREATE POLICY "Users can update their own cards"
ON cards FOR UPDATE
USING (auth.uid() = user_id);
```
