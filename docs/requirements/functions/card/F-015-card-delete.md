# F-015: カード削除

| 項目 | 内容 |

| 関連US | F-015 |

## 概要

ユーザーが不要になったカードを削除できるようにする。削除は論理削除ではなく物理削除とし、関連データ（タグ紐付け、学習ログ）も削除される。

## ユーザーフロー

1. **[トリガー]** ユーザーがカード編集画面またはホーム画面で「削除」をタップ
2. **[システム]** 削除確認ダイアログを表示
3. **[ユーザー]** 「削除する」ボタンをタップ
4. **[システム]** カードと関連データを削除
5. **[完了]** 削除完了メッセージを表示、ホーム画面に戻る

## 入力/出力

| 種別 | 項目名 | 型 | 必須 | 説明 | 制約・バリデーション |
|-----|-------|---|-----|-----|-------------------|
| 入力 | cardId | string | Yes | 削除対象カードID | UUID形式 |
| 出力 | success | boolean | - | 削除成功フラグ | - |

## ビジネスルール

| ID | ルール | 条件 | 結果 |
|----|-------|-----|-----|
| BR-F015-01 | 物理削除 | カード削除時 | DBからレコードを完全削除 |
| BR-F015-02 | カスケード削除 | カード削除時 | card_tags, review_logsも削除 |
| BR-F015-03 | 所有者チェック | 他ユーザーのカード | 削除不可（404エラー） |
| BR-F015-04 | 確認必須 | 削除操作 | 必ず確認ダイアログを表示 |

## エラーケース

| ID | 条件 | エラーメッセージ | 対処 |
|----|-----|----------------|-----|
| E-F015-01 | カード不在 | カードが見つかりません | ホーム画面に戻る |
| E-F015-02 | 権限なし | このカードを削除する権限がありません | ホーム画面に戻る |
| E-F015-03 | 削除失敗 | カードの削除に失敗しました。再度お試しください | リトライ |

## 受け入れ条件（AC）

- [ ] AC-01: カードを削除できること
- [ ] AC-02: 削除前に確認ダイアログが表示されること
- [ ] AC-03: 削除後、ホーム画面からカードが消えていること
- [ ] AC-04: 他ユーザーのカードは削除できないこと
- [ ] AC-05: 削除をキャンセルした場合、カードは維持されること

## 画面要件

### 削除確認ダイアログ

- 「このカードを削除しますか?」メッセージ
- カードのテキストプレビュー（識別用）
- 「この操作は取り消せません」警告文
- 「キャンセル」ボタン
- 「削除する」ボタン（赤色、危険操作を示す）

## 技術仕様

### Supabase連携

```typescript
// カード削除（カスケード削除はDB側で設定）
const { error } = await supabase
  .from('cards')
  .delete()
  .eq('id', cardId)
  .eq('user_id', userId)
```

### DB設計（カスケード削除）

```sql
-- card_tags のカスケード削除
ALTER TABLE card_tags
ADD CONSTRAINT fk_card_tags_card
FOREIGN KEY (card_id) REFERENCES cards(id)
ON DELETE CASCADE;

-- review_logs のカスケード削除
ALTER TABLE review_logs
ADD CONSTRAINT fk_review_logs_card
FOREIGN KEY (card_id) REFERENCES cards(id)
ON DELETE CASCADE;
```

### RLS

```sql
CREATE POLICY "Users can delete their own cards"
ON cards FOR DELETE
USING (auth.uid() = user_id);
```
