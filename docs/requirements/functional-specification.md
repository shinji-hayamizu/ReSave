## 1. SRSアルゴリズム仕様

### 1.1 カードの状態

| 状態 | DB値 | 説明 | 表示場所 |
|------|------|------|----------|
| **未学習** | `new` | 一度も学習していないカード | 未学習タブ |
| **復習中** | `active` | 復習サイクル進行中 | 復習中タブ |
| **完了** | `completed` | 復習終了（習得済み） | 完了タブ |

### 1.2 自己評価（3段階）

| 評価 | DB値 | 意味 | 動作 |
|------|------|------|------|
| **もう一度** | `again` | 思い出せなかった | currentStep=0にリセット |
| **OK** | `ok` | 考えて思い出せた | 次のステップへ進む |
| **覚えた** | `remembered` | 完全に覚えた | 即座に完了状態へ |

### 1.3 復習間隔（デフォルト）

| ステップ | 次回復習 | 備考 |
|----------|----------|------|
| 1 | 1日後 | 初回または覚え直し後 |
| 2 | 3日後 | |
| 3 | 7日後 | |
| 4 | 14日後 | |
| 5 | 30日後 | |
| 6 | 180日後 | 半年 |
| → | **完了** | ステップ6でOK評価後、自動的に完了 |

### 1.4 カスタマイズ機能

- ユーザーが復習間隔を自由に設定可能（ReviewScheduleテンプレート）
- ステップ数の増減も可能
- カード作成時にテンプレートの間隔がカードにコピーされる

### 1.5 状態遷移図

```
[new] ──OK評価──→ [active: step 1]
                       │
                       ├─「もう一度(again)」──→ [active: step 0] (リセット)
                       │
                       ├─「OK」──→ [active: step 2] → ... → [active: step 5]
                       │                                          │
                       │                                          └─「OK」──→ [completed]
                       │
                       └─「覚えた(remembered)」──→ [completed]
```

### F-001: ユーザー新規登録

#### 概要
- 目的: ユーザーがアカウントを作成し、学習データを保存できるようにする
- 関連US: US-001

#### ユーザーフロー
1. [トリガー] ユーザーが「新規登録」ボタンをタップ
2. [システム] 登録フォームを表示
3. [ユーザー] メールアドレス、パスワード、パスワード確認を入力
4. [システム] 入力値をバリデーション
5. [システム] 確認メールを送信
6. [ユーザー] メール内のリンクをクリック
7. [完了状態] アカウントが有効化され、ログイン状態になる

#### 入力/出力

| 種別 | 項目名 | 型 | 必須 | 説明 | 制約・バリデーション |
|------|--------|-----|------|------|---------------------|
| 入力 | email | string | ○ | メールアドレス | 有効なメール形式、最大255文字、重複不可 |
| 入力 | password | string | ○ | パスワード | 最小8文字、英数字混合必須 |
| 入力 | passwordConfirm | string | ○ | パスワード確認 | passwordと一致 |
| 出力 | user | User | - | 作成されたユーザー | - |

#### 受入条件
- [ ] AC-01: 有効なメールアドレスとパスワードで登録できること
- [ ] AC-02: 既に登録済みのメールアドレスではエラーが表示されること
- [ ] AC-03: パスワードが8文字未満の場合エラーが表示されること
- [ ] AC-04: パスワードと確認が一致しない場合エラーが表示されること
- [ ] AC-05: 確認メールが送信されること

#### 画面モック要件
- 入力フィールド: メールアドレス、パスワード、パスワード確認
- 登録ボタン
- ログインページへのリンク
- エラーメッセージ表示エリア

#### 【要確認】事項
- [ ] メール認証は必須か、オプションか

### F-023: 次回復習日の自動計算

#### 概要
- 目的: 忘却曲線に基づいて最適な復習タイミングを自動設定する
- 関連US: US-023

#### ロジック詳細

```
入力: 現在のステップ(currentStep), 評価(assessment), スケジュール(schedule)
出力: 新しいステップ(newCurrentStep), 次回復習日(nextReviewAt), ステータス(status)

if assessment == 'again':
    newCurrentStep = 0
    nextReviewAt = 今日 + schedule[0]日
    status = 'active'

elif assessment == 'ok':
    nextStep = currentStep + 1
    if nextStep >= schedule.length:
        status = 'completed'
        nextReviewAt = null
        completedAt = NOW()
    else:
        newCurrentStep = nextStep
        nextReviewAt = 今日 + schedule[currentStep]日
        status = 'active' (newカードの場合はnew→activeに変更)

elif assessment == 'remembered':
    status = 'completed'
    completedAt = NOW()
    nextReviewAt = null
```

#### デフォルト間隔設定

```javascript
const defaultIntervals = {
  1: 1,    // 1日後
  2: 3,    // 3日後
  3: 7,    // 7日後
  4: 14,   // 14日後
  5: 30,   // 30日後
  6: 180   // 180日後
};
```

#### 受入条件
- [ ] AC-01: 評価に応じて正しい次回復習日が設定されること
- [ ] AC-02: ステップ6でOK評価後、自動的に完了になること
- [ ] AC-03: カスタム間隔設定が反映されること

### F-050: データ同期

## 3. 機能間の依存関係

```
F-001 (ユーザー登録)
  └→ F-002 (ログイン)
      └→ F-050 (データ同期)
          │
          ├→ F-013 (カード作成)
          │   ├→ F-014 (カード編集)
          │   ├→ F-015 (カード削除)
          │   └→ F-016 (タグ付け) <-> F-017 (タグ管理)
          │         └→ F-018 (タグフィルタ)
          │
          ├→ F-020 (今日の復習確認)
          │   └→ F-021 (カード学習)
          │       └→ F-022 (自己評価)
          │           └→ F-023 (次回復習日計算)
          │
          └→ F-030 (日別統計)
```
