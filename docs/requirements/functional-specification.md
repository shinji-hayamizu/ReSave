## 1. SRSアルゴリズム仕様

### 1.1 カードの状態

| 状態 | 説明 | 表示場所 |
|------|------|----------|
| **新規** | 未学習のカード | 学習タブ |
| **学習中** | 復習サイクル進行中 | 学習タブ |
| **完了** | 復習終了（習得済み） | 完了タブ |

### 1.2 自己評価（3段階）

| 評価 | 意味 | 動作 |
|------|------|------|
| **覚え直し** | 思い出せなかった | ステップ1にリセット |
| **OK** | 考えて思い出せた | 次のステップへ進む |
| **覚えた** | 完全に覚えた | 即座に完了状態へ |

### 1.3 復習間隔（デフォルト）

| ステップ | 次回復習 | 備考 |
|----------|----------|------|
| 1 | 1日後 | 初回または覚え直し後 |
| 2 | 3日後 | |
| 3 | 7日後 | |
| 4 | 14日後 | |
| 5 | 30日後 | |
| 6 | 180日後 | 半年 |
| → | **完了** | ステップ6でOK評価後、自動的に完了 |

### 1.4 カスタマイズ機能

- ユーザーが復習間隔を自由に設定可能
- ステップ数の増減も可能（最小2ステップ、最大10ステップ）
- デッキごとに個別設定可能

### 1.5 状態遷移図

```
[新規] ──学習開始──→ [学習中: ステップ1]
                           │
                           ├─「覚え直し」──→ [学習中: ステップ1] (リセット)
                           │
                           ├─「OK」──→ [学習中: ステップ2] → ... → [学習中: ステップ6]
                           │                                              │
                           │                                              └─「OK」──→ [完了]
                           │
                           └─「覚えた」──→ [完了]
```

### F-001: ユーザー新規登録

#### 概要
- 目的: ユーザーがアカウントを作成し、学習データを保存できるようにする
- 関連US: US-001

#### ユーザーフロー
1. [トリガー] ユーザーが「新規登録」ボタンをタップ
2. [システム] 登録フォームを表示
3. [ユーザー] メールアドレス、パスワード、パスワード確認を入力
4. [システム] 入力値をバリデーション
5. [システム] 確認メールを送信
6. [ユーザー] メール内のリンクをクリック
7. [完了状態] アカウントが有効化され、ログイン状態になる

#### 入力/出力

| 種別 | 項目名 | 型 | 必須 | 説明 | 制約・バリデーション |
|------|--------|-----|------|------|---------------------|
| 入力 | email | string | ○ | メールアドレス | 有効なメール形式、最大255文字、重複不可 |
| 入力 | password | string | ○ | パスワード | 最小8文字、英数字混合必須 |
| 入力 | passwordConfirm | string | ○ | パスワード確認 | passwordと一致 |
| 出力 | user | User | - | 作成されたユーザー | - |

#### 受入条件
- [ ] AC-01: 有効なメールアドレスとパスワードで登録できること
- [ ] AC-02: 既に登録済みのメールアドレスではエラーが表示されること
- [ ] AC-03: パスワードが8文字未満の場合エラーが表示されること
- [ ] AC-04: パスワードと確認が一致しない場合エラーが表示されること
- [ ] AC-05: 確認メールが送信されること

#### 画面モック要件
- 入力フィールド: メールアドレス、パスワード、パスワード確認
- 登録ボタン
- ログインページへのリンク
- エラーメッセージ表示エリア

#### 【要確認】事項
- [ ] メール認証は必須か、オプションか

### F-023: 次回復習日の自動計算

#### 概要
- 目的: 忘却曲線に基づいて最適な復習タイミングを自動設定する
- 関連US: US-023

#### ロジック詳細

```
入力: 現在のステップ(currentStep), 評価(rating)
出力: 新しいステップ(newStep), 次回復習日(nextReviewDate)

if rating == '覚え直し':
    newStep = 1
    nextReviewDate = 今日 + intervals[1]

elif rating == 'OK':
    if currentStep == maxStep:
        status = '完了'
        nextReviewDate = null
    else:
        newStep = currentStep + 1
        nextReviewDate = 今日 + intervals[newStep]

elif rating == '覚えた':
    status = '完了'
    nextReviewDate = null
```

#### デフォルト間隔設定

```javascript
const defaultIntervals = {
  1: 1,    // 1日後
  2: 3,    // 3日後
  3: 7,    // 7日後
  4: 14,   // 14日後
  5: 30,   // 30日後
  6: 180   // 180日後
};
```

#### 受入条件
- [ ] AC-01: 評価に応じて正しい次回復習日が設定されること
- [ ] AC-02: ステップ6でOK評価後、自動的に完了になること
- [ ] AC-03: カスタム間隔設定が反映されること

| 入力 | period | string | - | 表示期間 | 'week' / 'month' / 'year' |
| 出力 | stats | DailyStats[] | - | 日別統計 | - |

#### 受入条件
- [ ] AC-01: 過去7日間の学習カード数が表示されること
- [ ] AC-02: 月間表示に切り替えられること
- [ ] AC-03: 学習した日が視覚的に区別できること

#### 画面モック要件
- カレンダービュー（学習した日をハイライト）
- 棒グラフ（日別カード数）
- 今日の学習カード数

### F-050: データ同期

#### 概要
- 目的: 複数デバイス間でデータをリアルタイム同期する
- 関連US: US-050

#### ユーザーフロー
1. [トリガー] ユーザーがデータを変更（カード作成、学習完了等）
2. [システム] 変更をサーバーに送信
3. [システム] 他デバイスに変更を反映
4. [完了状態] 全デバイスで同一のデータが表示される

#### 同期対象データ
- デッキ（作成、編集、削除）
- カード（作成、編集、削除）
- 学習進捗（ステップ、次回復習日）
- 設定（復習間隔カスタマイズ）

#### 受入条件
- [ ] AC-01: オンライン時、変更が即座に同期されること
- [ ] AC-02: オフライン時、変更がローカルに保存されること
- [ ] AC-03: オフライン→オンライン復帰時、自動的に同期されること
- [ ] AC-04: 競合が発生した場合、最新の変更が優先されること

#### 【要確認】事項
- [ ] オフライン対応の優先度（MVP vs 後続フェーズ）
