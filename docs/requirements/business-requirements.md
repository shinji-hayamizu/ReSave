# ReSave ビジネス要件定義書

## 1. プロジェクト概要

### プロジェクト名
**ReSave** - 忘却曲線記憶カードアプリ

### 一言サマリー（エレベーターピッチ）
ReSaveは、エビングハウスの忘却曲線に基づいた間隔反復システム（SRS）を採用した記憶カードアプリです。科学的に最適化されたタイミングで復習通知を送り、最小限の学習時間で長期記憶への定着を実現します。Web・モバイル両対応でいつでもどこでも学習可能です。

### ターゲットユーザー
- 資格試験・受験勉強中の学生・社会人
- 語学学習者（単語・フレーズの暗記）
- 専門知識を効率的に習得したいビジネスパーソン
- 生涯学習に取り組む一般ユーザー

### 解決する課題
| 課題 | 詳細 |
|------|------|
| 非効率な復習 | 何度も同じ内容を復習しているが、タイミングが悪く記憶に定着しない |
| 学習の継続困難 | いつ何を復習すべきか分からず、モチベーションが低下する |
| デバイス間の分断 | PCで作成したカードをスマホで復習できない |
| 進捗の不可視化 | 自分の学習状況が把握できず、達成感を得にくい |

## 2. 背景・目的

### なぜこのサービスを作るのか
従来の暗記学習は「繰り返し回数」に依存していたが、認知科学の研究により「復習のタイミング」が記憶定着の鍵であることが証明されている。忘却曲線に基づいた最適なタイミングで復習を促すことで、同じ学習時間でも記憶効率を飛躍的に向上させることができる。

### 現状の課題
1. **既存アプリの複雑さ**: Anki等の既存SRSアプリは機能が多く、初心者には敷居が高い
2. **クロスプラットフォーム対応の不足**: Web/モバイル間でシームレスに同期できるアプリが少ない
3. **日本語UIの不足**: 主要なSRSアプリは英語UIが中心で、日本のユーザーに馴染みにくい

### 期待される効果
- 学習効率の向上（従来比2-3倍の記憶定着率）
- 学習習慣の定着（毎日の復習リマインダー）
- ストレスフリーな学習体験（シンプルなUI/UX）

## 3. 機能要件（ユーザーストーリー形式）

### 認証・ユーザー管理

| ID | ユーザーストーリー | 優先度 |
|----|-------------------|--------|
| F-001 | ユーザーとして、メールアドレスとパスワードで新規登録したい。学習データを保存するため。 | Must |
| F-002 | ユーザーとして、登録したアカウントでログインしたい。デバイス間でデータを同期するため。 | Must |
| F-003 | ユーザーとして、パスワードをリセットしたい。パスワードを忘れた場合に対応するため。 | Must |
| F-004 | ユーザーとして、Googleアカウントでログインしたい。登録の手間を省くため。 | Should |
| F-005 | ユーザーとして、プロフィール（表示名・アバター）を編集したい。自分らしくカスタマイズするため。 | Could |

### カード・デッキ管理

| ID | ユーザーストーリー | 優先度 |
|----|-------------------|--------|
| F-010 | ユーザーとして、新しいデッキ（カードの集まり）を作成したい。学習内容を分類するため。 | Must |
| F-011 | ユーザーとして、デッキの名前・説明を編集したい。整理しやすくするため。 | Must |
| F-012 | ユーザーとして、デッキを削除したい。不要になったデッキを整理するため。 | Must |
| F-013 | ユーザーとして、カード（表面：質問、裏面：答え）を作成したい。暗記したい内容を登録するため。 | Must |
| F-014 | ユーザーとして、カードの内容を編集したい。間違いを修正するため。 | Must |
| F-015 | ユーザーとして、カードを削除したい。不要なカードを整理するため。 | Must |
| F-016 | ユーザーとして、カードを別のデッキに移動したい。整理し直すため。 | Should |
| F-017 | ユーザーとして、カードにタグを付けたい。横断的に検索・フィルタするため。 | Could |
| F-018 | ユーザーとして、カードに画像を添付したい。視覚的な記憶を強化するため。 | Could |

### 学習・復習機能

| ID | ユーザーストーリー | 優先度 |
|----|-------------------|--------|
| F-020 | ユーザーとして、今日復習すべきカードを確認したい。効率的に学習するため。 | Must |
| F-021 | ユーザーとして、カードの表面を見て答えを思い出し、裏面で答え合わせしたい。自己テストするため。 | Must |
| F-022 | ユーザーとして、カードの記憶度を自己評価（忘れた/曖昧/覚えている/完璧）したい。次回の復習タイミングを最適化するため。 | Must |
| F-023 | ユーザーとして、忘却曲線に基づいて次回復習日が自動設定されてほしい。最適なタイミングで復習するため。 | Must |
| F-024 | ユーザーとして、新規カードを一定枚数ずつ学習に追加したい。学習ペースを調整するため。 | Should |
| F-025 | ユーザーとして、学習セッションの時間制限を設定したい。集中力を維持するため。 | Could |

### 進捗・統計

| ID | ユーザーストーリー | 優先度 |
|----|-------------------|--------|
| F-030 | ユーザーとして、日別の学習カード数を確認したい。学習習慣を把握するため。 | Must |
| F-031 | ユーザーとして、デッキごとの習熟度を確認したい。苦手分野を把握するため。 | Should |
| F-032 | ユーザーとして、連続学習日数（ストリーク）を確認したい。モチベーションを維持するため。 | Should |
| F-033 | ユーザーとして、学習履歴をグラフで可視化したい。成長を実感するため。 | Could |

### 通知・リマインダー

| ID | ユーザーストーリー | 優先度 |
|----|-------------------|--------|
| F-040 | ユーザーとして、復習すべきカードがある時に通知を受けたい。復習を忘れないため。 | Should |
| F-041 | ユーザーとして、通知の時間帯を設定したい。生活リズムに合わせるため。 | Should |
| F-042 | ユーザーとして、通知をオフにしたい。集中したい時に邪魔されないため。 | Should |

### データ同期・エクスポート

| ID | ユーザーストーリー | 優先度 |
|----|-------------------|--------|
| F-050 | ユーザーとして、Web/モバイル間でデータが自動同期されてほしい。どのデバイスでも学習を継続するため。 | Must |
| F-051 | ユーザーとして、デッキをCSV/JSON形式でエクスポートしたい。バックアップや他アプリ移行のため。 | Could |
| F-052 | ユーザーとして、CSV/JSON形式でカードをインポートしたい。大量のカードを効率的に登録するため。 | Could |

## 4. ビジネスリスク

| ID | リスク | 影響度 | 発生確率 | 対策 |
|----|--------|--------|----------|------|
| BR-001 | 競合アプリ（Anki, Quizlet等）との差別化が不明確 | 高 | 中 | シンプルなUI/UXと日本語ローカライズで差別化。初心者向けのオンボーディングを充実 |
| BR-002 | ユーザーの継続率が低い（学習習慣が定着しない） | 高 | 高 | ストリーク機能、リマインダー通知、進捗可視化でモチベーション維持を支援 |
| BR-003 | モバイルアプリのストア審査が通らない | 中 | 低 | Apple/Google のガイドラインを事前に確認し、準拠した設計を行う |
| BR-004 | サーバーコストが想定以上に増加 | 中 | 中 | 初期はサーバーレス構成（Vercel + Supabase等）でスモールスタート |
| BR-005 | 個人情報漏洩によるレピュテーションリスク | 高 | 低 | 認証はOAuth推奨、パスワードは適切にハッシュ化、通信はHTTPS必須 |

## 5. 用語定義

| 用語 | 定義 |
|------|------|
| カード | 表面（質問）と裏面（答え）で構成される暗記用の単位 |
| デッキ | カードの集まり。学習テーマごとに分類 |
| 忘却曲線 | エビングハウスが提唱した、時間経過と記憶保持率の関係を示す曲線 |
| 間隔反復（SRS） | Spaced Repetition System。記憶度に応じて復習間隔を調整する学習法 |
| ストリーク | 連続して学習した日数。継続のモチベーション指標 |
| 習熟度 | カードごとの記憶定着度合い。復習回数と正答率から算出 |
