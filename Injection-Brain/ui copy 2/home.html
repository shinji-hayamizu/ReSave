useStateとuseEffectの違いは?

0/6
復習日 2025/12/24

useState: コンポーネントの状態を管理するHook。状態の初期値を設定し、状態を更新する関数を提供します。

useEffect: 副作用を処理するHook。コンポーネントのレンダリング後に実行され、データの取得、購読、DOM操作などに使用します。

// useState example
const [count, setCount] = useState(0);

// useEffect example
useEffect(() => {
  document.title = `Count: ${count}`;
}, [count]);

ジェネリクスの主な用途は?

1/6
復習日 2025/12/25

ジェネリクスは型の再利用性を高めるための機能です。

  • 型安全性を保ちながら柔軟なコードを書ける
  • コードの重複を減らせる
  • コンパイル時に型チェックができる
// Generic function
function identity<T>(arg: T): T {
  return arg;
}

// Generic interface
interface Box<T> {
  value: T;
}

const stringBox: Box<string> = { value: "hello" };
const numberBox: Box<number> = { value: 42 };

プリペアドステートメントとは?

2/6
復習日 2025/12/26

プリペアドステートメントは、SQLクエリを事前にコンパイルし、パラメータを後からバインドする仕組みです。

SQLインジェクション攻撃を防ぐための重要なセキュリティ対策です。

// Bad: SQL Injection vulnerable
const query = `SELECT * FROM users WHERE id = ${userId}`;

// Good: Prepared Statement
const stmt = db.prepare('SELECT * FROM users WHERE id = ?');
stmt.run(userId);

コンテナとイメージの違いは?

2/6
復習日 2025/12/27

イメージ: アプリケーションと依存関係をパッケージ化した読み取り専用のテンプレート。設計図のようなもの。

コンテナ: イメージから作成される実行中のインスタンス。イメージを元に起動した実際の環境。

# イメージをビルド
docker build -t myapp:latest .

# イメージからコンテナを起動
docker run -d --name myapp-container myapp:latest

HTTPメソッドの使い分けは?

2/6
復習日 2025/12/28
  • GET: リソースの取得（読み取り専用）
  • POST: 新規リソースの作成
  • PUT: リソースの完全な更新/置換
  • PATCH: リソースの部分的な更新
  • DELETE: リソースの削除
GET    /users      # ユーザー一覧取得
GET    /users/1    # 特定ユーザー取得
POST   /users      # 新規ユーザー作成
PUT    /users/1    # ユーザー情報更新
DELETE /users/1    # ユーザー削除

ブランチの作成とマージの方法は?

3/6
復習日 2025/12/29

Gitブランチは独立した開発ラインを作成し、後でメインラインに統合できます。

# ブランチの作成と切り替え
git checkout -b feature/new-feature

# 作業後、コミット
git add .
git commit -m "Add new feature"

# mainブランチに戻る
git checkout main

# ブランチをマージ
git merge feature/new-feature

# ブランチの削除
git branch -d feature/new-feature

justify-contentとalign-itemsの違いは?

4/6
復習日 2025/12/30

justify-content: 主軸（デフォルトは横方向）に沿った配置を制御

align-items: 交差軸（デフォルトは縦方向）に沿った配置を制御

.container {
  display: flex;

  /* 横方向の配置 */
  justify-content: center;  /* 中央揃え */
  justify-content: space-between; /* 両端揃え */

  /* 縦方向の配置 */
  align-items: center;  /* 中央揃え */
  align-items: flex-start; /* 上揃え */
}
