# 並列サブエージェント構築テンプレート生成

指定されたspecのtasks.mdを解析し、並列実行可能なグループ分けとサブエージェント起動命令を生成します。

## 使用方法

```bash
/dev/parallel-build <spec-name>
```

## 引数

$ARGUMENTS

---

## 実行手順

### Step 1: Spec検索

以下のパスを順に検索:
1. `.kiro/specs/{spec-name}/tasks.md`
2. `.claude/specs/{spec-name}/tasks.md`

### Step 2: タスク解析

tasks.mdから以下を抽出:
- `- [ ] X.Y タスク名` → 未完了タスク
- `(P)` マーク → 並列実行可能
- メジャー番号でグループ化

### Step 3: 依存関係分析

- 順次実行が必要なタスク（Backend系: DB → API → 検証）
- 並列実行可能なタスク（UI系: 各コンポーネント独立）
- 依存関係のあるタスク間の順序制約

### Step 4: エージェントグループ生成

以下の形式でグループ分けを提案:

```
グループ分け:
- Agent1(Backend): タスクX.1→X.2→X.3を順番に（各タスク完了時に個別コミット）
- Agent2: タスクY.1 コンポーネントA
- Agent3: タスクY.2 コンポーネントB
- Agent4: タスクY.3 コンポーネントC
```

```markdown
以下テンプレートでcc-sddを実行する命令を作って。
並列サブエージェントで実装して。

グループ分け:
- Agent1(Backend): タスクX.1→X.2→X.3を順番に（各タスク完了時に個別コミット）
- Agent2: タスクY.1 [タスク名]
- Agent3: タスクY.2 [タスク名]
- Agent4: タスクY.3 [タスク名]
- Agent5: タスクZ.1 [タスク名]
- Agent6: タスクZ.2 [タスク名]

## コミットルール
各サブエージェントは担当タスク完了時に必ずコミットすること。

コミットメッセージ規約を必ず遵守すること

Agent1(Backend)はX.1, X.2, X.3それぞれで個別コミット。

既存の型定義とパターンを踏襲すること。think hard
```

---

## グループ分けの原則

### Backend系（順次実行）
- DB/マイグレーション → API/Server Actions → バリデーション/検証
- 依存関係があるため1エージェントで順次実行
- 各サブタスク完了時に個別コミット

### UI系（並列実行）
- 独立したコンポーネントは別エージェントで並列実行
- 共通コンポーネントへの依存がある場合は順序制約を追加

### テスト系
- 対象コードと同じエージェントに含めるか、別エージェントで後から実行

```bash

```

既存の型定義とElectronパターンを踏襲すること。think hard

---

## 注意事項

- エージェント数は5-8程度が最適（多すぎるとコンテキスト競合）
- 共有リソース（型定義、共通コンポーネント）へのアクセスに注意
- 競合が予想される場合は順次実行グループにまとめる
