## 🎯 目的（Claude Code向け前提）

* Claude Code による **Reactコードの自動レビュー・修正提案・指摘の一貫性**を担保する
* React の **レンダー最適化・中断・再実行**を前提とした安全な実装を強制する
* Hooks の誤用による **潜在バグ・将来不具合を事前に排除**
* 長期運用・複数リポジトリ前提

---

## 🔧 基本方針（最重要）

* **React コンポーネントおよび Hooks は必ず Pure（純粋）に保つ**
* **レンダー中に副作用を発生させない**
* **Hooks の呼び出し順序は常に固定**
* **React の管理外でコンポーネントや Hooks を実行しない**

---

## 🧼 レンダー純粋性ルール（必須）

### 基本

* **同じ props / state / context なら必ず同じ UI を返す**
* レンダー中に以下を行うことは禁止：
  * 通信 / fetch / API 呼び出し
  * 購読・イベント登録
  * analytics / logging / tracking
  * `Date.now()` / `Math.random()` など非決定的処理
  * DOM の直接操作
  * グローバル変数・モジュールスコープ変数の更新
* レンダー中に **外部で生成された値** を変更しない

### イミュータブル制約

* props / state を直接変更しない
* 関数引数として受け取った object / array / Map / Set を破壊的に変更しない
* JSX に渡した値を「後から変更される前提」で扱わない

---

## 🔁 副作用の配置ルール

* **ユーザー操作由来の副作用** → イベントハンドラに記述
  * 例：クリック時の保存、送信、遷移、トラッキング
* **レンダー後に同期すべき副作用** → `useEffect` に記述
  * 例：購読、タイマー、外部同期、DOM反映
* **レンダー中に副作用があるコードは即NG**

---
